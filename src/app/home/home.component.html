Welcome to C.R.E.A.M. Finance's User Dashboards!

Before you dive in, here's a brief overview:

  • I built an Angular dashboard detailing metrics across all of Cream's money markets as well as an aggregated dashboard for key metrics that iceCREAM holders might care about. Each page details a specfic money market and includes both current and historical data for the following metrics:
    • Total Value Locked
    • Total Value Supplied
    • Total Value Borrowed
    • Utilization Rate
    • Loan Origination (interest earned for lenders)
    • Loan Revenue (protocol revenue)

  • As this is a hackathon project, I cut quite a few corners to make sure I delivered a 'finished' product on time. Should you want to implement these dashboards in a production environment I'd like to improve on what you see today to provide an excellent user experience (loading speed, accuracy, etc).

  • There are bugs. I wasn't able to squash them all before my submission. One that I'd like to specifically mention is that all statistics use the current price for individual assets, which may skew data (especially historical data). Using historical asset prices is something I'd want to implement for production use and is a feature I have implemented in the past for 88mph's stats page.

  • As part of my project, I developed a subgraph to index basic data for the Cream token and Cream token holders (including iceCREAM). Feel free to fork or use this subgraph for your own needs. https://thegraph.com/legacy-explorer/subgraph/0xszeth/cream-token

  • Also, data does take a few seconds to load, especially for the chains that Cream has been active on for a long time. If it seems like it's taking a while to load (more than ~5 seconds) just give the page a refresh or check the console for errors.

  • This was an opportunity for me to try some new things, so I did! This was my first time using ethers.js and chainlink price feeds just to name a couple.

  • I had an awesome time building this, so thank you for the opportunity to compete on this project. I've learned a lot and believe I have progressed as a developer in the DeFi space.
